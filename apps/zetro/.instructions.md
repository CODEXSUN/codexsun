# Zetro: Local AI Code Assistant with Memory

Zetro is a comprehensive local AI code assistance system owned by `apps/zetro`. It provides intelligent planning, code generation, execution, and debugging through the Orexso API, with persistent project memory and fast context retrieval.

## Quick Start

### Prerequisites

1. **Orexso API** running on `http://localhost:6005`
   ```bash
   curl http://localhost:6005/health
   ```

2. **Node.js / TypeScript** environment
   ```bash
   npm install
   npm run build
   ```

### Basic Usage

#### Via VS Code Agent

1. Open VS Code with this workspace
2. Activate Zetro commands from `apps/zetro/`:
   - `Zetro: Plan` - Analyze code without changes
   - `Zetro: Write` - Generate code preview
   - `Zetro: Execute` - Apply changes
   - `Zetro: Fix` - Find and fix errors
   - `Zetro: Chat` - Interactive conversation
   - `Zetro: Memory Status` - Project memory overview

#### Via TypeScript API

```typescript
import { createAgentWithMemory } from "./apps/zetro/src/agentic-tools-memory";

const agent = await createAgentWithMemory(process.cwd());

// Plan phase
const plan = await agent.plan("Create a validation utility");
console.log(plan.plan.steps);

// Memory available
agent.memory       // Project memory manager
agent.cache        // Context cache
agent.quickRef     // Codebase quick reference
agent.index        // Repository index
```

## Architecture

### Ownership Boundaries

All Zetro code lives within `apps/zetro/`:

```
apps/zetro/
├── src/
│   ├── orexso-client.ts              # Orexso API client
│   ├── agentic-tools.ts              # Core workflow tools
│   ├── agentic-tools-memory.ts       # Memory integration factory
│   ├── project-memory.ts             # Project memory manager
│   ├── codebase-analyzer.ts          # Repository indexer
│   ├── execution-history.ts          # Execution tracking
│   ├── context-cache.ts              # In-memory cache layer
│   ├── quick-reference.ts            # Quick-ref generator
│   ├── memory-boundary.ts            # .ai folder setup
│   └── examples.ts                   # Usage examples
├── tests/
│   └── agentic-tools.test.ts         # Test suite
├── .agent.md                         # Agent manifest
├── .instructions.md                  # User instructions
├── AI_GUIDANCE.md                    # Orexso API docs
└── README.md                         # This file
```

### Memory Structure

All AI memory is stored in the project root `.ai/` folder (externally):

```
.ai/
├── memory/
│   ├── projects/                    # Project metadata
│   ├── executions/                  # Execution history
│   ├── patterns/                    # Learned patterns
│   └── context/                     # Session/task context
├── cache/
│   └── cache.json                   # Persistent cache
├── quick-ref.md                     # Architecture summary
└── config.json                      # Memory configuration
```

The `.ai/` folder is added to `.gitignore` to prevent committing AI memory.

## Components

### OrexsoClient (`orexso-client.ts`)
Low-level API bindings for all Orexso endpoints:
- Chat and streaming
- Structured agent tasks
- File operations
- Task interruption and rollback
- MCP integration

### AgenticTools (`agentic-tools.ts`)
High-level workflow API implementing discover-plan-execute-verify:
- **Plan Mode**: Analyze code without changes
- **Write Mode**: Generate code preview
- **Execute Mode**: Apply changes with snapshots
- **Fix Mode**: Identify and fix errors
- **Chat Mode**: Stream-based conversation
- Streaming support with partial tokens
- Session and context management
- Rollback and interruption handling

### ProjectMemory (`project-memory.ts`)
Persistent project-level memory manager:
- Execution history tracking
- Learned patterns storage
- Context key-value store (scoped: global/session/task)
- Metadata about project learning progress
- TTL-based context expiration

### CodebaseAnalyzer (`codebase-analyzer.ts`)
Fast codebase indexer:
- Scans `apps/` directory structure
- Detects languages, frameworks, ownership
- Maps files to apps
- Generates architecture summaries
- Tracks patterns (API routes, components, services, hooks)

### ContextCache (`context-cache.ts`)
In-memory cache with disk persistence:
- Fast lookups for frequently used data
- Automatic TTL expiration
- Autosave to disk every 5 seconds
- Survives across sessions

### QuickReference (`quick-reference.ts`)
Auto-generated codebase knowledge:
- Architecture summary markdown
- App inventory
- Framework and language overview
- Cached for fast retrieval

### AgentWithMemory Factory (`agentic-tools-memory.ts`)
Creates a fully integrated agent with memory:

```typescript
const agent = await createAgentWithMemory(projectPath);
// agent.tools        - AgenticTools instance
// agent.memory       - ProjectMemoryManager
// agent.cache        - ContextCache
// agent.quickRef     - String with architecture
// agent.index        - RepositoryIndex
```

## Workflows

### 1. Plan, Write, Execute Loop

```typescript
const agent = await createAgentWithMemory(process.cwd());

// 1. Plan: Analyze impact
const plan = await agent.plan("Create authentication middleware");
console.log("Plan steps:", plan.plan.steps);

// 2. Write: Generate code
const code = await agent.write("Implement the middleware");
console.log("Preview:", code.generatedCode);

// 3. Execute: Apply changes
const result = await agent.execute("Apply middleware to API");
console.log("Applied:", result.appliedChanges);

// Memory automatically records all steps
```

### 2. Error Fixing

```typescript
const agent = await createAgentWithMemory(process.cwd());

// Analyze and fix errors
const fixes = await agent.fix("Fix TypeScript compilation errors");
console.log("Proposed fixes:", fixes.generatedCode);

// Track execution
const recentExecutions = await agent.memory.getRecentExecutions(5);
```

### 3. Context-Aware Chat

```typescript
for await (const chunk of agent.chat("Explain the auth flow")) {
  process.stdout.write(chunk);
}
```

## Performance

- **Small tasks** (< 10 files): 5-15 seconds
- **Medium tasks** (10-50 files): 15-30 seconds
- **Large tasks** (50+ files): 30-120 seconds
- **Streaming**: Real-time token delivery
- **Cache hits**: < 100ms

First request may take longer as model warms up.

## Troubleshooting

### Orexso Not Running

```bash
# Check API
curl http://localhost:6005/health

# Check port
lsof -i :6005  # macOS/Linux
netstat -ano | findstr :6005  # Windows
```

### Authentication Issues

```bash
# Set API key
export OREXSO_API_KEY=your-key
```

### Model Warm-up

```bash
curl http://localhost:6005/health | jq .warmup_status
# Wait for warmup_ready: true
```
